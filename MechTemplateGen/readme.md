This is a fairly simple tool used to generate and name mechdef, chassisdef, movedef, icon and itemcol files to easily create new variants and mechs for Battletech mods
It is designed to allow for rapidly generating the files letting the user focus more of performing individual fix ups for anything that isn't handled by the tool.
It can set fixed equipment, inventory equipment, armor, max armor, base tonnage (for mods not using MechEngineer), max move speed, tags, hardpoints, itemcol
assignment for mech and mechparts, yangs thoughts, details, stockrole, heatsinks, prefab and hardpoint data links and properly handle variant and file naming.
The tool has no error checking for most of its settings, so it will not check if you are assigning more armor than the chassis can take or putting too many objects
into an inventory section.

To use the tool, either open the MechTemplateGen.slnx file and then run it via visual studio, or copy the exe from bin\Debug\net10.0 into the MechTemplateGen folder
and then run it directly.

this tool should work with any modpack that still uses mechdefs and chassisdefs, but if you encounter issues with using it then you can file a ticket (you will need
to indicate which modpack you were using and provide a config file that can reproduce the issue).  Note, generating extra mech files will prevent you from asking
roguetech for support so do not bother them if you generate something with this and it breaks something.


# Important files
The important files for running this program (other than the exe) are the engineweightmapping.json, internalstructurepartmapping.json, itemmapping.json,
movedef_template.json and most importantly the main mechinfo.json file itself.

- engineweightmapping.json maps from the engine rating values to weights and gyro weights for regular and xl engines. 
- internalstructurepartmapping.json is used for computation of armor and structure maximum values, note, this tool uses the standard vanilla maximum value calculations
  i.e. TT values x 5 for structure + 1 for the head and x2 structure for front torso armor and full structure for rear armor (which means total torso armor max is 1.5x
  the TT maximum). If you don't want the tool to generate max armor values for your generated mechs, you can instead use a base chassisdef template file that corresponds
  to a mech of matching tonnage and thus already has the correct armor and structure values.
- itemmapping.json is used to allow for mapping from named used to specify fixed and inventory equipment and the actual item id and item type. if you run the tool without
  this file existing then the tool will automatically scan the mod and battletech data folders to find equipment files for building this file (NOTE, THIS CAN TAKE MANY
  MINUTES TO COMPLETE, BUT ONLY NEEDS TO BE DONE ONCE)
- movedef_template.json is used as the base template for movedef generation, if you do not want the tool to generate movedef files, then do not specify all of prefabName,
  mechTonnage and mechEngine for a mechinfo object (mechengine is only needed for mods that are not using mechengineer so for most large modpacks it will not generate
  movedef files)
- mechinfo.json is the primary data file, this contains an array (the surrounding [ and ] at the top and bottom of the file) that contains a sequence of mechinfo objects 
  (the data surrounded by { } in the file) separated by , see below for the formatting of a mech info object


# MechInfo object format
The MechInfo objects are each surrounded by {} in the mechinfo.json file and separated by , each field in them is standard json key value pairs so the field name should
be surrounded with "" marks and followed by : and then the field value should be separated from the : by at least one space
